# Feature Specification: Exhaustive React UI Parity (FR-008 & beyond)

Ця специфікація детально описує кожен елемент, кожну колонку та кожну кнопку, що існували в оригінальному (старому) статичному UI (в папці `src/arbitrator/presentation/static`), і гарантує, що новий React UI буде реалізовано зі 100% функціональним паритетом.

Всі числа, відсотки, кнопки та віджети повинні бути перенесені "один в один".

## 0. Правила Форматування (Formatting Rules)
Щоб кожна цифра і символ відображалися ідентично старому UI, усі React компоненти повинні використовувати утиліти форматування, що повторюють логіку старого `format.js`:

- **Пусті або невалідні значення (`null` / `undefined`):** Повинні відображатися як `"—"` (тире) або `"N/A"` (залежно від контексту).
- **Звичайні числа (`fmtNum`):** Заокруглюються до `4` знаків після коми (toFixed(4)) за замовчуванням.
- **PnL та Відсотки (`fmtPnl`, `fmtPercentDeposit`, `fmtStrategyProfit`):**
  - Обов'язковий знак перед числом (`+` для додатних, `−` (U+2212, не дефіс) для від'ємних).
  - Заокруглюються до `2` знаків після коми.
  - Усі відсоткові значення (наприклад, спреди) повинні мати знак `%` в кінці без пробілу (напр., `+1.23%`).
- **Кольорове кодування (CSS 클래си `pos`, `neg`, `na`):**
  - Додатні значення: клас `.pos` (колір `var(--green)`).
  - Від'ємні значення: клас `.neg` (колір `var(--red)`).
  - Порожні/N/A значення: клас `.na` (колір `var(--muted)`).
- **Великі об'єми (K/M) (`compactK`):**
  - `> 1,000,000`: `M` (напр., `1.23M`).
  - `> 1,000`: `K` (напр., `45K`, без знаків після коми).
  - Менше 1000: ціле число (без знаків після коми).

## 1. Налаштування (Settings Page)
**Мета**: Управління ключами API бірж та глобальними параметрами стратегій.
**Джерела даних**:
- WebSocket `/ws/settings` (отримання статусу бірж).
- REST API `GET /api/config/strategy` (отримання параметрів стратегій).
- REST API `PUT /api/config/strategy` (збереження параметрів стратегій).

### 1.1 Блок "Біржі" (Exchange API Keys)
Для кожної підтримуваної біржі виводиться:
- **Заголовок**: Назва біржі (напр., `MEXC API key`) та текст `(налаштовано)`, якщо ключ вже є в системі.
- **Поле вводу "API Key"**: Текстове поле (заповнюється замаскованим ключем `api_key_masked`, якщо є).
- **Поле вводу "API Secret"**: Поле типу `password`.
- **Поле вводу "API Password"**: Поле типу `password` (відображається лише для бірж, які цього вимагають через прапорець `has_password`, напр., Bitget).
- **Кнопка "Зберегти"**: Відправляє `settings.save_exchange` через WebSocket. Змінені значення секрету і пароля передаються лише якщо вони введені, інакше передаються пусті рядки `""`.

### 1.2 Блок "Стратегії" (Strategy Configuration)
Усі параметри, що повертаються бекендом через `/api/config/strategy`, повинні бути:
- Згруповані по вкладках (Tabs) або візуальних блоках відповідно до їх категорій:
  - "Live Auto-Trade", "Paper Auto-Trade", "Live Захисти", "Paper Захисти", "Движок Стратегій", "Модуль Opportunity", "Історичний моніторинг", "Інші налаштування", "Advanced (JSON)".
- Кожен параметр повинен мати:
  - Зрозумілий Label (переклад українською/англійською згідно зі словником `STRATEGY_META`).
  - Підпис (Desc) - дрібним текстом пояснення логіки параметру з `STRATEGY_META`.
  - Поле вводу (Input) - числове або текстове; для boolean - Checkbox (Enabled/Disabled).
  - "Advanced (JSON)" - велике текстове поле (textarea) для редагування масивів/об'єктів `strategy_overrides` та `allowed_strategies` у форматі JSON.
- **Кнопка "Зберегти зміни"**: З'являється при редагуванні параметрів та відправляє об'єкт із всіма змінами через `PUT /api/config/strategy`.

## 2. Скрінер (Live Screener Page)
**Мета**: Відображення арбітражних можливостей у реальному часі.
**Джерело даних**: WebSocket `/ws/screener` (`screener.snapshot` та `screener.delta`).

### 2.1 Верхня панель (Filters)
- **Status Badge**: "Connecting...", "Disconnected" або статус з `screener.snapshot`.
- **Лічильник**: Відображає "Відфільтровано / Всього пар".
- **Поле "Min 24h vol (K USDT)"**: Input. Значення за замовчуванням має братися з поля `filters.min_volume_k_usdt` першого снапшоту від бекенду.
- **Поле "Min spread %"**: Input. Значення за замовчуванням має братися з `filters.min_spread_pct`.
- **Кнопка "Apply to Server"**: Відправляє повідомлення `screener.set_filter` у WebSocket.

### 2.2 Таблиця (Screener Data Table)
Колонки (строго без горизонтального скролу, текст має переноситися `wrap-break-word`):
1. **Asset**: Назва пари (напр. DOGE/USDT).
2. **Біржі (MEXC, BITGET, GATE, BINGX)**: Кожна колонка містить дві ціни одна під одною:
   - "F: {ціна}" (Futures)
   - "S: {ціна}" (Spot)
3. **Max P**: Максимальна ціна серед усіх (формат 4 знаки).
4. **Min P**: Мінімальна ціна серед усіх (формат 4 знаки).
5. **Spread**: Відсоток спреду (жирним шрифтом, зелений клас `.pos` для +, червоний клас `.neg` для -).
6. **Delta**: Зміна спреду (з кольоровим кодуванням).
7. **Vol(K)**: Об'єм (форматується через функцію `compactK`).
8. **Профіти по стратегіях**: (futures_futures, futures_spot_2ex, futures_spot_1ex, funding_ff, funding_fs, funding_diff_dates).
9. **Кнопка "Open"**: Відкриває сторінку Opportunity у новій вкладці з підставленими параметрами.

## 3. Сторінка Opportunity
**Мета**: Детальний аналіз та виконання угод для конкретної пари.
**Джерело даних**: WebSocket `/ws/opportunity?symbol={}&short={}&long={}`.

### 3.1 Верхня панель та параметри
- Input "Symbol", "Short Exchange", "Long Exchange" та кнопка "Load Data".
- Бейджі поточних бірж ("S: binance", "L: mexc").

### 3.2 Таблиця стратегій (Strategy Calculations)
Колонки: Strategy, Spread (%), Delta, Fee %, Max Vol (макс об'єм для входу), Details (помилки або "ОК").
- **Кнопки дій (Action Buttons):** Якщо стратегія має "OK" у колонці Details, навпроти неї має з'являтися кнопка виконання (наприклад, "Execute" або іконка), яка ініціює відкриття позиції по вибраній стратегії (надсилає подію `opportunity.action`).

### 3.3 Панель управління (Trading Actions)
Кнопки для управління відкритою позицією (відправляють `opportunity.action` з відповідними параметрами):
- "Добрати" (Accumulate)
- "Закрити" (Close Partial)
- "Закрити всі позиції" (Close All)

### 3.4 Стакани ордерів (Order Books)
Дві картки (для Short та Long біржі):
- **Заголовок**: Назва біржі та поточний "Best Ask" (для short) або "Best Bid" (для long).
- **Таблиця стакану**:
  - Аски (червоні, відсортовані знизу вверх, найдешевший ближче до центру).
  - Спредовий розділювач (лінія).
  - Біди (зелені, відсортовані зверху вниз, найдорожчий ближче до центру).
  - Колонки: "Price" та "Volume".

### 3.5 Графік Opportunity (Opportunity Chart)
**Критично важливий компонент.** Повинен відтворювати функціонал файлу `opportunity_chart.js`:
- Візуалізує розбіжність цін (спред) та самі ціни `Bid` та `Ask` бірж з часом.
- Бібліотека для чартів має підтримувати серії даних: Spread, Short Exchange Price (Bid/Ask), Long Exchange Price (Bid/Ask).
- Графік має оновлюватися в реальному часі на основі повідомлень з WebSocket.

## 4. Сторінки Ордерів та Демо Торгів (Orders & Paper Trades)
**Мета**: Відображення відкритих та закритих позицій (реальних або віртуальних).
**Джерела даних**: `/ws/orders` та `/ws/paper_trades`.

### 4.1 Панель Summary
Відображає сумарні показники:
- Total PnL (формат PnL із `+`/`−` та кольором `.pos`/`.neg`).
- Fees (Комісії).
- Funding (Фандінг).
- Кількість Open / Closed ордерів.
- Фільтри (кнопки перемикачі): "All", "Open", "Closed" (повинні реально фільтрувати відображуваний список груп у таблиці нижче).

### 4.2 Таблиця Ордерів
Кожен рядок — це "група" (ордер, що складається з кількох ніг).
Колонки Групи:
- Кнопка розгортання (▶ / ▼).
- **Asset**: Назва активу.
- **Exchanges**: Бейджі S:{біржа} та L:{біржа}.
- **Status**: "open" або "closed".
- **Opened**: Час відкриття (формат: дата і час).
- **Spread**: Показує "In: {x}%" та "Out: {y}%".
- **Fees**: Сума комісій.
- **Funding**: Сума фандінгу.
- **PnL**: Загальний PnL групи (з кольоровим кодуванням).

### 4.3 Розгорнута секція (Legs)
При натисканні на ▶ розгортається таблиця "ніг" (конкретних угод на біржах):
- **Leg**: Напрямок (Short/Long) + Біржа + Плече (Leverage).
- **Volume**: Об'єм угоди.
- **Entry**: Ціна входу.
- **Exit**: Ціна виходу.
- **Fees**, **Funding**, **PnL** для конкретної ноги (з кольоровим кодуванням).

## 5. Історія Скрінера (Historical Monitors)
**Мета**: Пошук історичних можливостей та запуск автоматичних моніторів (Live Cards).
Це найскладніший модуль.

**Джерело даних**: `/ws/historical_screener`.

### 5.1 Верхня панель (Controls & Filters)
- **Time Window (seconds)**: Поле вводу (default: 3600).
- **Min Spread %**: Поле вводу (default: 1.0).
- **Min 24h Vol (USDT)**: Поле вводу (default: 0).
- Кнопки **Start Monitoring** та **Stop Monitoring** (відправляють команди `start` / `stop`).
Зміна будь-якого поля автоматично відправляє `update_filters`.

### 5.2 Таблиця Знайдених Можливостей (Opportunities Table)
Відмальовується з `msg.data.opportunities`.
Колонки:
- **Symbol**: Назва.
- **Exchanges**: (Short/Long).
- **Funding Rate**: Фандінг шорт та лонг.
- **Next Funding**: Таймер до фандінгу.
- **Funding Spread**: Відсоток (Current Spread).
- **Price**: Ціни.
- **Volume USDT ↕**: Об'єм 24h (формат через `compactK`).
- **Action**: Включає дві кнопки (функціонал як у `monitors.js`):
  1. **"Copy to Form"**: Копіює параметри цієї можливості у форму (в Live Monitor Card, якщо вона існує).
  2. **"Fast Trade"**: Швидка торгівля. Клік додає новий монітор, негайно передаючи налаштування для старту торгівлі.

### 5.3 Сітка Моніторів (Monitors Grid & Live Cards)
Сітка (CSS Grid), що відображає активні відстеження з `msg.data.monitors`. Оновлюється в реальному часі через `msg.data.live_state`.

**Структура "Live Card" (одна картка на кожен монітор)**:
- **Кнопка "X" (Close)**: Закриває монітор (відправляє `remove`).
- **Заголовок**: Зірочка, SYMBOL, піктограма булавки. Біржі: `EXCHANGE1 ↓` та `EXCHANGE2 ↑`.
- **Блок "Strategy Params"** (Усі параметри мають поля вводу (input) і галочку (✓), яка блимає при збереженні. Валідація не допускає відправку порожніх або нечислових значень там, де це не передбачено):
  - **Side**: Кнопки "Auto", "LONG", "SHORT".
  - **Open Spread**: Значення (%) та поле `T` (ticks).
  - **Close Spread**: Значення (%) та поле `T` (ticks).
  - **Order Size**: Input об'єму та відображення ≈ USD.
  - **Max orders**: Input (шт).
  - **Allowed size**: Текст (напр. "0/ Max:4000").
  - **Force Stop**: Checkbox.
  - **Total Stop**: Checkbox.
  - **Is Active?**: Кнопки "▶ Start", "■ Stop", "⟳ Restart". При кліку відправляють `is_active` в `update_config`.
- **Блок "Exchange Data Columns"** (дві колонки для двох бірж):
  - Funding rate.
  - Next Funding (таймер).
  - Ask / Bid.
  - Size.
  - Leverage.
  - Adjustment notification (кнопки "Notify only" / "Adjust").
  - Max size, Price, P/L, Realized PNL, Enter spread, Orders.
- **Блок "Spread Tracking"**:
  - Open spread: поточне значення, Min, Max.
  - Close spread: поточне значення, Min, Max.
  - Кнопка "Spread history".
  - Міні-графік (лінії Open/Close spread та осі).
