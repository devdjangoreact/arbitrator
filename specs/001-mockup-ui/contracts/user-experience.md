# User experience — що бачить користувач після імплементації фічі 001

**Аудиторія**: оператор арбітражу, локальний браузер.  
**Запуск**: `poetry run python main.py` → `http://127.0.0.1:8000`

Усі **видимі дані** на екранах приходять з **backend через WebSocket** (не зашиті в HTML).
HTML/CSS — лише каркас; числа, таблиці, графік, стакани оновлюються з push snapshot-ів.

---

## Два режими — що відчувається однаково / по-різному

| Аспект | `mock_data` (default) | `live` |
|--------|----------------------|--------|
| Вигляд UI | Як `maket/index.html` | Те саме |
| Ціни, спред, графік, стакани | Backend push, анімація | Реальні біржі (watch_*) |
| Таблиця стратегій, PnL у Orders | Розраховані / симульовані | Реальні позиції + PnL |
| «Добрати» / «Закрити» | In-memory: обʼєм і PnL змінюються в UI | Реальні ордери на біржах |
| Авто добір / авто скидання | Прапорці зберігаються; **авто-дії симулюються** простою логікою mock | `OpportunityAutoEvaluator` + `ArbitrageMonitorWorker` |
| API keys Settings | In-memory (до рестарту) | Запис у runtime / `.env` |

## Сценарій: перший запуск (`mock_data`)

### 1. Відкриває браузер

- Бічна панель: Screener, Opportunity, Orders · **2**, Settings.
- Активний екран **Screener**.

### 2. Screener (живий стіл моніторингу)

**Бачить:**

- Рядок статусу: `Streaming: 5 symbols · Exchanges: mexc, bitget, gate, bingx · …`
- Таблицю: DOGE, SOL, XRP, ADA, TON — ціни fut/spot по біржах, Max/Min, Spread %, Δ, Vol K.
- 6 колонок прибутку по стратегіях (Ф-Ф … Ф різн.) — числа з backend.
- Кнопку **Open Opportunity** в кожному рядку.

**Поведінка:**

- Кожну ~1 с ціни й спред **плавно змінюються** (mock tick).
- Зміна **Min spread** → WS-команда → таблиця фільтрується.
- **Reconnect** → короткий статус connecting → знову потік даних.
- **Open Opportunity** на DOGE → перехід на Opportunity з MEXC short / BingX long.

### 3. Opportunity (робоче місце по одному символу)

**Бачить (ліва колонка):**

- Topbar: `DOGE/USDT`, бейджі S·MEXC, L·BingX.
- Дві картки бірж: баланс, фандінг %, countdown, плече (select), комісії, лічильник ордерів.
- Таблицю **6 стратегій** — спред, ціни, комісії, фандінг, обʼєм, плече, заробіток, витрати, **чистий прибуток**.
- Блок **Параметри**: стратегія (select), набрано обʼєм, обʼєм до добору, пороги спреду, добір/закриття, авто-прапорці.
- Панель **Ордери DOGE/USDT** (відкриті/закриті) з PnL по ногах.

**Бачить (права колонка):**

- Графік 4 ліній (fut/spot × 2 біржі) — рухається в реальному часі.
- 4 стакани (fut + spot на кожній біржі) — ціни, обʼєми, fill-bar.

**Що може зробити (логіка):**

| Дія | Що відбувається в UI |
|-----|---------------------|
| Обрати **стратегію** в select | `opportunity.set_params` → активна стратегія в snapshot; таблиця стратегій підсвічує контекст |
| Змінити пороги спреду / обʼєм до добору | Зберігається в params на backend |
| Увімкнути **Авто добір** | Прапорець у snapshot; у mock — при spread ≥ порогу mock може сам додати обʼєм; у live — `OpportunityAutoEvaluator` |
| Увімкнути **Авто скидання** | Аналогічно для закриття за порогом закриття |
| Ввести 50 USDT → **Добрати** | `action_result` success → **набрано обʼєм** ↑, open order group ↑, PnL перераховується |
| **Закрити** partial | Обʼєм ↓, можливий partial у legs |
| **Закрити всі позиції** | Група → Closed, зникає з «Відкриті», лічильник Orders ↓ |
| Змінити **плече** | Оновлення на картці біржі + у strategy rows |

**Важливо:** немає окремої кнопки «Запустити стратегію». «Запуск» = обрати стратегію + ручний добір **або** увімкнути авто-режими.

### 4. Orders (глобальна історія)

**Бачить:**

- Заголовок: `2 відкритих арбітражі`, `2 open · 14 closed`.
- Фільтри: Усі / Відкриті / Закриті.
- Таблицю груп: asset, стратегія, біржі, дати, плече, обʼєм, вхід/вихід, комісія, фандінг, **PnL**, статус.
- Розгортання групи → 2 ноги (short/long) з детальним PnL.
- Кнопку **Opportunity** на відкритих групах → повернення на екран можливості.

**Після закриття позиції на Opportunity:**

- Група переходить у **Closed**, PnL і `closed_at` заповнюються.
- Sidebar `Orders · N` оновлюється.

### 5. Settings

**Бачить:** поля API key по біржах (masked).

**Робить:** вводить key/secret → Save → `configured: true`, masked key у полі.

У **mock** — до рестарту процесу; у **live** — ключі для реальної торгівлі.

---

## Сценарій: `live` + API keys

Те саме UI, але:

- Screener — реальні ticker з бірж.
- Opportunity — реальні баланси, books, chart з order book WS.
- **Добрати** → `OpportunityAccumulateService` → ордери на біржах.
- **Закрити** → `ArbitrageCloseService`.
- PnL у Orders — з позицій / `ArbitragePnlService`, не симуляція.
- **Авто добір/скидання** — фонові worker-и перевіряють спред і виконують дії (існуючий application layer).

Фаза live wiring: tasks **T034–T036** (після перевірки UI на mock).

---

## Що НЕ входить у фічу 001 (користувач цього не побачить)

- Окремий екран «історія стратегій» — прибуток по стратегіях видно в **Screener** (колонки) і **Opportunity** (таблиця 6 рядків), закриті угоди — в **Orders**.
- Deploy / Connect кнопки зі старих макетів — немає.
- Мобільна версія / multi-user.
- Гарантія прибутку в mock — числа демонстраційні.

---

## Підсумок одним реченням

Після запуску користувач отримує **повноцінний робочий інтерфейс як у макеті**: усі цифри з backend по WebSocket, можна фільтрувати скринер, відкрити opportunity, **обрати стратегію, добирати/закривати обʼєм, закрити всі позиції, дивитися PnL** по відкритих і закритих арбітражах; у режимі **mock** це симуляція з правдоподібною логікою, у **live** — справжня торгівля через наявні application services.
